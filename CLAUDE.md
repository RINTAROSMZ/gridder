# CLAUDE.md - AR空間グリッド検証アプリ

## プロジェクト概要

実空間の立体形状（道路、壁など）をリアルタイムに認識し、歩幅の引き込み効果を検証するためのWebARアプリケーション。

## アーキテクチャ & 技術スタック

- **フロントエンド (Web):** HTML5, WebXR Device API / Three.js (iOS Safari/Chrome 動作想定)。GitHub Pages (`git.io`) でホスト。
- **メインロジック (Python):** 空間ジオメトリの計算、および時間的連続性を担保する平滑化フィルタ（移動平均/カルマンフィルタ等）のプロトタイプ・メインアルゴリズムは**Python**で記述。
- **ラッパー & 高速化 (C++ / Wasm):** Pythonで書かれたロジックを**C++ライブラリ（pybind11、またはEmscriptenのPython環境へのブリッジ）でラップ**し、WebAssembly (Wasm) として一体化。ブラウザ上のJavaScriptから低遅延で呼び出せる軽量・高速な実行バイナリを生成する。

## 開発 & ビルドコマンド

_注意: PythonのWasmターゲット環境およびemsdkがセットアップされていることを確認してください。_

- **Wasm/Pythonビルド:** `emcc src/wrapper.cpp -O3 -s WASM=1 -s WITH_PYTHON=1 -o dist/core.js` （※環境に応じたビルドコマンドに調整）
- **ローカル開発サーバー起動:** `python3 -m http.server 8000`
- **コード整形:** `black internal/*.py` / `clang-format -i src/*.cpp`

## コードスタイル & ガイドライン

- **Python (メインロジック):** 配列演算は極力効率化し、コンパイル後のオーバーヘッドを減らす記述を意識する。
- **C++ (ラッパー):** Pythonコードへの値の受け渡し（JavaScript ⇄ C++ ⇄ Python）において、不要なコピーが発生しないよう参照やポインタを適切に管理する。
- **知覚的安定性（最重要）:** 認識した形状に対して毎フレームグリッドを再生成するのではなく、前フレームのデータを保持し、時間的に連続した一続きのオブジェクトとして滑らかに追従（Lerpやフィルタリング）させるロジックをPython側で実装する。
- **パフォーマンス:** iOS端末で60fpsを維持するため、JS-Wasm（Python）間のデータのやり取りは最小限の頂点バッファの更新に留める。
